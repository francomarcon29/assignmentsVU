Introduction

This is a website in which the user can find three main sections: navbar, body, and footer. First, the navbar sections presents a logo and three buttons. Each of them provides accessibility to the key parts of the main body which are the table of phones, specifications, and reviews. Moreover, the main body is formed by several parts, starting with a presentation banner, the table of phones, specifications, reviews, and the team. Diving more into details, the table of phones presents a dynamic functionality, where phones can be uploaded by completing the required fields at the bottom of the table. Therefore, these will be automatically updated in the table. To continue with the layout of the website, the specifications are displayed in four boxes vertically organized that represent different technical insights. After this, the reviews are shown in boxes as well but displayed horizontally. To sum up, the footer is where the information of the website is located, starting with the social media links that redirect to each platform, an embedded google map showing the location of the website “store”, and to conclude, the contact are where the user can email or call.

Accessibility highlights

1.1: Text Alternatives

We think this guideline is very important to our page because the presence of the images, especially the team images, would be missing completely to someone who can't see the images

2.1: Keyboard Accessible

This guideline is important because there are (especially in the form) a lot of buttons and text input fields that are core functionalities of the page. Without good keyboard accessibility, these functionalities can not be used by a user that can't use a mouse.

Improvement plan (in descending order of importance)

Improvement Implementation
Convey the meaning of the input fields so that people who cannot see the input fields can still know why they are there Add a descriptive title attribute to every input tag
Convey the meaning of team images when the user cannot view the images Add descriptive alt attributes to the team img tags
Use smaller images so that they render nicely even without css Scale the images down to a better size for the web
There should be a text alternative to the 'mobile' ASCII art used in the banner Add an abbr tag with the meaning of the ASCII art so that people know what it is even if it doesn't render properly or when they can't see the ASCII art
The images do not scale to window size of browser at all Add more media queries that scale the images down using width and height css
We could implement a way (other than color change) to show the user that they are hovering over something Also make the text bold when hovering over items
